Vue2之数组

2022-01-19 PV:

前言

数组的索引和数组长度发生改变，Vue2检测不到变化。

Vue官方文档

由于 JavaScript 的限制，Vue 不能检测数组和对象的变化。

Vue 不能检测以下数组的变动：

  1. 当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue
  2. 当你修改数组的长度时，例如：vm.items.length = newLength

Vue监听Array变化

Vue2是通过Object.defineProperty实现响应式数据，是Object.defineProperty不能对数组已有元素实现监听吗？

可见：Object.defineProperty是可以对数组已有元素实现监听的。

只不过vue没有提供这一功能。

具体原因：

前面讲到，Object.defineProperty是可以对数组已有元素实现监听的。这里指的是能够检测索引的变化，但是对于数组元素的增加和删除是监听不到的。

在最前面的代码，新增数组元素，没有调用到Object.defineProperty的set()方法。

那vue是怎么实现的呢？

vue的解决方案：重写了数组的原型，更准确地说是拦截了数组的原型。

总结Vue监听Array步骤：

第一步：先获取原生 Array 的原型方法，因为拦截后还是需要原生的方法帮我们实现数组的变化。

第二步：对 Array 的原型方法使用 Object.defineProperty 做一些拦截操作。

第三步：把需要被拦截的 Array 类型的数据原型指向改造后原型。

解决方案

举个例子

修改数组，使用Vue.set和splice。

修改数组长度，使用 splice。

新增数组元素，还可以使用Vue.set，Vue.set(数组名，插入的索引下标值，插入的数据元素)。

删除数组元素， 还可以使用Vue.delete。
